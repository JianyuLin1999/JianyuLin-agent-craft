# Comment Voice — 评论语言纪律

写 PR / issue 评论时的措辞规范。跨多个评论类 skill 共享。

**核心一句话**:reviewer 的声音应该**像跟朋友说话**,不像老师批改作业。说事实、说自己的想法、给选择 —— 不评价对方、不下命令、不替人想。

---

## §1 三条核心原则

### 1. 说事实,别评价人

看到什么就说什么。区别在"主语":主语是事情,不是对方。

> ❌ "你没看这部分"<br>
> ✅ "这部分我没在文档里看到"

### 2. 说想法时,带上"我"

观点是 reviewer 自己的,不是绝对真理。明示"我"让对方知道这是 1 个意见,可以反驳。

> ❌ "应该改成 X"<br>
> ✅ "我想改成 X" / "我担心如果不改会 Y"

### 3. 给选择,别下命令

reviewer 不是决策者。提供路径,让对方挑。

> ❌ "你立刻做这三件事:1. X 2. Y 3. Z"<br>
> ✅ "两条路可以走:A 是快的,B 是稳的,你看"

---

## §2 两个对立的失败模式

LLM 容易在两端来回摔。纠正 A 容易过头滑到 B。

**A. 居高临下**:"你没看 X" / "你应该 Y" / "现在做的方向不对"
**B. 替人找借口**:"你大概是没时间细看" / "可能你不熟悉这块"

两个都错。**正确的不是找平衡,是完全跳出"评价对方"这件事** — 只说事情本身的状况,既不指责也不替人开脱。

替人找借口看起来礼貌,其实是另一种不尊重 — 等于"我替你想好了你为什么没做"。

---

## §3 致谢要能指着东西

夸的时候,问自己:**我能指着哪段具体的代码 / 文档 / 动作说这句话吗?**

指不到 → 是哄,删。

> ❌ "非常优秀的设计,继续保持"<br>
> ✅ "你这次加的那个测试,把一个之前没人测过的角覆盖了"

不要"先骂后哄"的对比结构(列一堆问题,结尾突然 "但你基本功很好")。哄人挽救不了前面的批评,只让两边都失真。

---

## §4 不替对方想

reviewer 最容易越界的四件事:

1. **预测对方的反应**("你看完会发现 X")— 让对方自己得出结论
2. **解释对方的行为**("你大概是因为 X")— 不解释原因,只说事情
3. **替对方拆活**("分成 5 个 PR:第一个做 X...")— "建议拆,具体怎么拆等我们对齐了再聊"
4. **替对方估工作量**("这件事很简单,几行代码")— 不估,只说"我看到要改的范围是 N 个地方"

底层一句话:**reviewer 给信号,对方做决策**。给得越多越像替人想,对方越容易防御。

---

## §5 字数和密度

写完看一眼字数。超出下面这个表,**默认有软话可以砍**:

| 评论类型 | 上限 |
|---|---|
| 同意通过 + 致谢 | 300 字 |
| 要求改一改 | 600 字 |
| 建议拆分 / 重构 | 700 字 |
| 先放一放 | 500 字 |

软话长什么样:
- "在我能给答复之前,有几件事我需要先跟你对齐" → 直接说"几件事"
- "你这份 X 很详细,看得出花了心思" → 删
- "我自己看了一遍,几个判断点想跟你确认" → 直接列判断

---

## §6 比喻能不用就不用

事实和表格已经把话讲清楚了,比喻就是冗余,删。

真要用比喻时,**不要让对方在比喻里变成"小白"**。让对方代入设计者 / 同等专业身份。

---

## §7 写完 draft 跑这 8 条(0 容忍)

| 句子像这样 | 改法 |
|---|---|
| 以 "你没 / 你应该 / 你必须 / 你不能" 开头 | 改成"我看到 X" 或 "我担心 X" + 给选择 |
| "立刻做这 N 件事:1. X 2. Y" 形态 | 改成 "N 条路你挑" |
| 出现 "你大概是 / 可能是因为 / 估计你..." | 删掉,只说事情 |
| "你看完会发现 X" / "你会注意到 Y" | 删掉,让对方自己得出结论 |
| "拆成 N 个 PR:第一个 X,第二个 Y..." | 改成 "建议拆,具体怎么拆对齐之后再说" |
| 列了一堆问题,结尾"但你基本功是好的" | 砍掉对比结构,要么纯正向收尾,要么不收尾 |
| 致谢里指不出来具体的代码 / 动作 / 文档 | 删,空夸比不夸更伤 |
| 总字数超过 §5 表里的上限 | 砍过渡 / 元话语 / 客套开头 |

## §使用方式

两种方式,优先选 1:

**方式 1(推荐):整段复制 §1-§7 进 skill prompt**

agent 在 prompt 加载时就拿到完整规则,不依赖跟链接 / 跟路径。集成成本 5 秒。

**方式 2:在 skill 里链接引用本文件**

适合仓库内运行的 skill。skill SKILL.md 加一句:"评论语言纪律见 `knowledge-assets/comment-voice.md`,自检必跑"。

### 本机镜像

本机 `~/.claude/skills/SHARED-COMMENT-VOICE.md` 跟本文件手动同步,或者用 symlink:

```bash
ln -sf <agent-craft-path>/knowledge-assets/comment-voice.md ~/.claude/skills/SHARED-COMMENT-VOICE.md
```
